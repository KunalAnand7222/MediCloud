1. PROJECT SUMMARY
================================================================================
MediCloud is a full-stack telemedicine/hospital management platform built with:
  - Frontend : React (Vite) + TailwindCSS
  - Backend  : Node.js + Express.js
  - Database : MongoDB (Mongoose ODM)
  - Realtime : Socket.IO (WebSockets)
  - Auth     : JWT (Access + Refresh Tokens) + Google OAuth

Roles supported: Admin, Doctor, Patient
Each role has its own dashboard, navigation, and access-controlled routes.

6. ROUTING & ROUTE PROTECTION (frontend/src/App.jsx)
================================================================================

  Route guards:
  - PublicRoute    → Redirects logged-in users away from login/register
  - ProtectedRoute → Redirects unauthenticated users to /login

9. BACKEND - MIDDLEWARE (backend/middleware/index.js)
================================================================================

  protect       → JWT verification middleware. Checks Bearer token or cookie,
                  validates JWT, checks user.isActive before granting access.

  authorize     → Role-based access control. Takes roles array, rejects if
                  req.user.role not in allowed list. (403 Forbidden)

  errorHandler  → Global error handler. Handles Mongoose CastError,
                  duplicate key (11000), ValidationError, JWT errors.

  upload        → Multer file upload middleware. Saves to uploads/problems/.
                  Accepts images only, max 5MB.

12. SECURITY FEATURES
================================================================================
  - JWT access tokens + refresh token rotation
  - HttpOnly cookies (fallback) + localStorage
  - Helmet.js (HTTP security headers)
  - CORS restricted to CLIENT_URL only
  - Rate limiting: 1000 req/15min general, 20 req/15min for auth routes
  - bcryptjs password hashing (12 rounds)
  - isActive check blocks deactivated users at middleware level
  - Role-based route guards on both frontend and backend
